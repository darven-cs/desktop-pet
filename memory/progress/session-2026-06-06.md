### 决策记录

| 决策 | 选项 | 理由 |
|---|---|---|
| AppError 类型 | **struct + 4 个 constructor** | enum + serde 序列化麻烦,struct 跟 TS 1:1 |
| 串行枚举命名 | `rename_all = "SCREAMING_SNAKE_CASE"` | 错误码 `E_XXX` 必须大写带前缀 |
| PNG 解析 | **手读 IHDR 前 24 字节** | 避免加 `image` crate 依赖 |
| 帧数算 | **sheet_w / sheet_h** | ffmpeg 抽帧会插重复帧，sheet 帧数才是播放真相 |
| menu 菜单尺寸 | **估算常量** | open() 时还没 render,只能估。02+ 可改成 measure 后调 |
| 切动画同步性 | **preload 所有 sheet + state 同步切** | R2 黑帧 < 50ms 只能这样,不用 'transitioning' 异步过渡 |
| 窗口尺寸 API | **LogicalSize 实例** | Tauri 2 不接受 `{ type: "Logical", w, h }` 字面量 |

## 续 06-07 凌晨：验收 + Bug 修复

### Bug 1: 精灵图动画不播放

- **现象**：所有动画的 sheet 图静止不动
- **根因**：`@keyframes play` 写在 `<style scoped>` 里，CSS 自定义属性 `var(--end-x)` 解析不到元素的行内 style
- **修复**：`@keyframes pet-play` 移到独立的非 scoped `<style>` 块
- **附带改动**：加 `:key="currentEntry.id"` 到 `.pet-sprite`，强制切换动画时重建 DOM

### Bug 2: 拉屎/思考动画黑帧多

- **现象**：poop 121 帧（源 36 帧）、think 26 帧（源 22 帧），大量插值黑帧
- **根因**：`gif-to-sheet.sh` 的 ffmpeg 命令缺 `-vsync 0`，按 GIF 声明帧率（50fps）插值
- **修复**：
  - 分解帧：`ffmpeg -y -vsync 0 -i "$INPUT" ...`
  - tile 拼合：改用分解帧文件 `$TEMP_DIR/frame_%04d.png` 而非重读 GIF
- **结果**：poop 121→36 帧、think 26→22 帧、touch_nose 28 帧无变化
