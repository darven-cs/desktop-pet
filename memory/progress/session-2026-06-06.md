## 决策记录

| 决策 | 选项 | 理由 |
|---|---|---|
| Spec 位置 | `docs/specs/编号+功能名.md` | 按实现顺序编号,方便回看 |
| 评审机制 | Claude 出意见 + 用户勾选 | 单人开发流里 SDD 评审的最低成本落地 |
| 冻结机制 | frontmatter `status` 字段 | 与文档同生共死,简单可追溯 |
| 强制级别 | **硬拦截** | 前期 30 分钟 spec 换后期少返工 |
| 动画列表来源 | **Rust IPC** (`list_animations`) | 扩展性强,02 改 impl 不改接口 |
| 右键菜单 UI | **绝对定位 `<div>`** | 最轻,200 行内可实现 |
| 状态机实现 | **self-rolled reducer + decider 独立模块** | decider 独立是为 02 LLM 替换铺路 |
| 4 个问题合并 | **方案 A:合并为 01 一个 spec** | 4 个问题共享"窗口尺寸自适应"和"动画状态机",拆开有大量重复 |
| 长期目标暴露 | **用户主动分享,我记入 vision 文档** | 后续 spec 都能从同一份路线图找方向 |
